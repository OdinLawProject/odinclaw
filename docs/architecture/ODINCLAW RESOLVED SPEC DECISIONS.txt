ODINCLAW RESOLVED SPEC DECISIONS
==================================================

INHERITED BASE VIEWS
--------------------------------------------------
- intelligence is architectural, not model-only
- governance before capability release
- continuity matters
- durable memory is not session recall
- trust and boundary handling are explicit
- repair and rollback are first-class
- symbiosis means interdependence with boundaries
- the system is receipt-native and auditable

MEMORY BOUNDARY
--------------------------------------------------
Shell memory:
- session-local operational context

ODIN memory:
- durable continuity
- doctrine
- trust history
- conflict memory
- durable lessons

RECEIPTS MUST EXIST FOR
--------------------------------------------------
- tool actions
- browser actions
- provider calls
- plugin actions
- durable memory writes
- trust-state changes
- approvals and denials
- rollback and repair events
